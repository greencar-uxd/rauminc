## 요약 (TL;DR)

세 시안 모두 **실행 완성도와 페이지 간 일관성은 시안 단계 기준으로 높습니다.** 디자인 토큰이 페이지별로 흐트러짐 없이 유지되고, 명도 대비는 본문 텍스트 기준 전 시안이 WCAG AA를 통과하며, 외부 요청 0건·CLS 방지(aspect-ratio)·reduced-motion 폴백 등 성능·모션 기본기가 잘 잡혀 있습니다. JS 에러는 18페이지 전부 0건.

다만 **세 시안이 같은 골격을 공유하기 때문에 공통 크리티컬 버그 6건이 18페이지 전체에 동일하게 존재합니다.** 특히 ① 모바일 메뉴를 열면 닫을 수 없는 z-index 버그, ② about 페이지 모바일 레이아웃 파괴(인라인 스타일이 미디어쿼리 무효화), ③ 메인 페이지의 "클릭되는 것처럼 보이지만 죽어 있는" 카드 UI는 **시안 시연 중에도 바로 드러나는 결함**이므로 프레젠테이션 전 수정을 권합니다.

## 1. 3안 공통 이슈 (18페이지 전체 해당)

세 시안은 공통 CSS/JS 골격을 공유하므로 아래 이슈는 **전 페이지에 동일하게** 존재합니다. 행 번호는 대표로 각 시안 메인 페이지 기준입니다.

### 🔴 Critical — 시안 승인과 별개로 반드시 수정

**C-1. 모바일 메뉴를 열면 닫을 수 없음** — `header{z-index:100}`이 스태킹 컨텍스트를 만들어 내부의 햄버거 버튼 `.mbtn{z-index:160}`이 컨텍스트에 갇히고, 전체화면 오버레이 `.mnav{z-index:150}`(header의 형제 요소)가 버튼 위를 완전히 덮습니다. Playwright 실측으로 확인: 메뉴 오픈 후 버튼 위치의 최상위 요소는 `div#mnav.mnav.open`이며 클릭이 오버레이에 가로채집니다. Escape 핸들러도 없어 링크로 페이지를 이동하는 것 외에 닫을 방법이 없습니다.
→ **수정**: 닫기 버튼을 `.mnav` 내부에 배치(또는 header z-index를 오버레이 위로) + `Escape` 키 핸들러 + `aria-expanded` 토글 + 메뉴 오픈 시 body 스크롤 잠금.

**C-2. about 페이지 모바일 레이아웃 파괴 (75px 가로 오버플로 실측)** — `a-about.html:278`, `b-about.html:285`, `c-about.html:279`의 `<div class="sp-blurbs rv" style="grid-template-columns:repeat(4,1fr)">`(PROCESS)와 인접한 `.sp-grid.four` 인라인 스타일(WHAT WE DO)이 900px 미디어쿼리의 1열 축소 규칙보다 항상 우선합니다. 390px 실측 결과 세 시안 모두 75px 가로 오버플로가 발생하고, 카드가 열당 약 70px로 짜부라져 "Hospita / Housin"처럼 글자가 잘리며 PROCESS 문단은 한 글자씩 줄바꿈됩니다(스크린샷 확보).
→ **수정**: 인라인 스타일 제거, `.sp-blurbs.cols4` 같은 클래스로 CSS에서 열 수 제어.

**C-3. 문의 폼 라벨이 입력과 프로그램적으로 미연결** — `*-contact.html`의 6개 필드 전부 `<label>성함 *</label><input>` 형태로 `for`/`id` 연결이 없어(레포 전체 `for=` 0건) 스크린리더가 필드명을 읽지 못합니다(WCAG 1.3.1/4.1.2). `name` 속성도 전무해 실제 제출 시 데이터가 전송되지 않으며, 연락처 필드에 `type="tel"`·`autocomplete`가 없습니다. 체크박스만 라벨 래핑으로 정상.
→ **수정**: 전 필드 `for`/`id` + `name` + `autocomplete="name|organization|tel|email"` + `type="tel"`. 제출 알림 `.sp-alert`에 `role="status"`(aria-live) 부여.

**C-4. 메인 페이지의 죽은 클릭 UI (가짜 어포던스)** — 메인의 프로젝트 카드(`.card`)와 카테고리 행(`.cat-row`)이 `cursor:pointer` + 호버 화살표/밑줄 애니메이션까지 갖췄지만 `<div>`라서 클릭해도 아무 일도 없고 Tab 포커스도 불가합니다. 동일 외형의 카드가 work 페이지에서는 `<a>`로 동작해 페이지 간 시맨틱까지 불일치합니다.
→ **수정**: 카드는 `*-project.html`, 카테고리 행은 `*-work.html`로 링크화.

**C-5. JS 실패 시 본문 대부분이 영구 비표시** — `.rv{opacity:0}`이 CSS 기본값이고 IntersectionObserver가 `.in`을 붙여야 보이는 구조인데 `<noscript>` 폴백이 0건입니다. JS 차단/실패 시 제목·카드·폼까지 빈 화면으로 남습니다(reduced-motion 사용자만 CSS로 구제).
→ **수정**: `<noscript><style>.rv{opacity:1;transform:none}</style></noscript>` 또는 `html.js` 클래스 게이팅.

**C-6. 히어로 캐러셀 제어 접근성** — 자동 전환(6초)을 멈추는 수단이 호버뿐이라 터치·키보드 사용자는 정지 불가(WCAG 2.2.2). A안은 컨트롤 자체가 없고, B안 도트·C안 번호는 포커스 불가능한 `<i>` 요소에 click 핸들러만 달려 있습니다. C안 비활성 번호는 실측 대비 2.07:1로 UI 컴포넌트 기준 3:1 미달.
→ **수정**: `<button aria-label="슬라이드 N">`로 교체 + 44px 히트 영역 + 일시정지 수단. (참고: `prefers-reduced-motion` 시 자동재생을 시작하지 않는 처리는 세 시안 모두 올바르게 구현되어 있음 — 좋은 부분.)

### 🟡 Warning

**W-1. 헤딩 계층 오류** — about·recruit 페이지에 **h1이 2개**(하단 CTA도 h1), work·contact는 h1→h3으로 h2 건너뜀, 전 페이지 푸터가 h4로 점프. CTA는 h2로 정리.
**W-2. 유효하지 않은 HTML** — work·about의 카드 캡션이 `<span class="sp-cap">` 안에 `<h3>`를 포함(인라인 요소 안 블록 헤딩, 스펙 위반). `sp-cap`을 `div`로.
**W-3. 닫힌 모바일 메뉴가 Tab 순서에 잔류** — `.mnav`가 `opacity:0;pointer-events:none`일 뿐 `visibility:hidden`이 아니라, 데스크톱 포함 모든 상태에서 보이지 않는 링크 5개가 포커스를 받습니다. `visibility:hidden` 또는 `inert` 토글 추가.
**W-4. 터치 타깃 44px 미달** — 햄버거(실측 약 42×26px), 필터 탭(약 35–37px), 채용 APPLY 버튼(약 38–40px), 헤더 nav 링크(약 21px).
**W-5. 한글 어중 줄바꿈** — `word-break:keep-all` 미적용으로 모바일에서 제목이 단어 중간에서 꺾입니다(실측: B안 스테이트먼트 "디테일로" → "디/테일로"). 제목·리드 계열에 `word-break:keep-all` 권장.
**W-6. 스테이트먼트 섹션 좌우 패딩 소실 (CSS 셀렉터 충돌)** — `<div class="wrap statement">`에서 뒤에 선언된 `.statement{padding:140px 0}`이 `.wrap`의 좌우 48px 패딩을 0으로 덮어씁니다. 실측: A·B안 데스크톱에서 본문 최대 폭 텍스트가 **화면 왼쪽 끝(0px)에 밀착**. C안은 중앙정렬(`margin:0 auto`)로 데스크톱에선 가려지지만 980px 이하에서 전폭으로 붙습니다. frontend-design 스킬이 경고하는 전형적 패턴("`.section`류 클래스 결합 시 패딩 상쇄 주의").
→ **수정**: `.statement{padding:140px 48px}` 또는 wrap 내부에 statement를 중첩.
**W-7. `height:100vh` 히어로** — 모바일 주소창 변동으로 하단 SCROLL 힌트·진행바가 가려질 수 있음. `100dvh/svh` 병기 권장.
**W-8. 내비게이션 상태 표시 부재** — 현재 페이지 하이라이트(`aria-current`) 없음, 스킵 링크 없음, 필터 탭에 `aria-pressed` 없음(호버와 활성 스타일이 동일해 오독 유발), 햄버거에 `aria-expanded` 없음.
**W-9. 본문 폰트 16px 미달 전면적** — 본문 13–15px, 마이크로 라벨 8–11px. 특히 폼 입력이 14px라 iOS에서 포커스 시 자동 확대(줌) 유발 — 폼 입력만이라도 16px로.
**W-10. reduced-motion 부분 누락** — `.rv`·슬라이드·티커는 처리됐으나 서브페이지 카드 호버 줌(`.sp-art`), 카테고리 행 패딩 애니메이션(레이아웃 속성!), `html{scroll-behavior:smooth}`, `header{transition:.4s}`는 감속 모드에서도 동작.

### 🔵 개선 제안 (시안 단계 허용, 개발 이관 시 반영)

- **이미지 시맨틱 훅 부재**: `<img>` 0건, 전부 CSS 그라디언트 placeholder. 실사진 교체 시 `<img loading="lazy" alt width height>` 슬롯 구조가 필요한데 현재 `.sp-art`가 `<span>`이라 마크업 수술이 필요. ※ placeholder임을 ※주석으로 명시한 규율은 우수.
- **메타 부재**: `meta description`·OG 태그·favicon 없음(시안이므로 허용, 오픈 전 필수).
- **CSS 220여 행이 파일마다 통짜 복제**: 아임웹 이관 시 공통 스타일시트로 추출. 동일 그라디언트가 `.a1–.a6`/`.g1–.g6` 두 벌로 정의된 것도 통합 대상.
- **메타 카피 누출**: work 페이지 방문자용 리드에 관리자 안내문("게시판형 구조로 내부에서 직접 추가·수정…"), contact 알림에 구축 플랫폼("아임웹") 노출 — ※주석 스타일로 분리.
- **표기 확인**: "Ochang **Entral** Park"가 6곳 반복 — 발주처 공식 표기("Central"?) 재확인 필요. A안 project 페이지 "템플릿로"→"템플릿으로" 조사 오탈자.
- **project 페이지 PREV/NEXT가 자기 자신 링크** — 시안 한계로 주석 고지되어 있어 적절히 처리됨.
